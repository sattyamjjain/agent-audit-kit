# AgentAuditKit v0.3.9 — release notes

**Released:** 2026-04-28
**Tag:** `v0.3.9`
**Rule count:** 180 (was 175)
**Test count:** 869 (was 830)
**Predecessor:** [v0.3.8](https://github.com/sattyamjjain/agent-audit-kit/releases/tag/v0.3.8) (2026-04-27)

## Headline

**Project Deal economic drift** (Anthropic 2026-04-26, $2.68/item
across model tiers) + **LangGraph ToolNode regression**
(prebuilt 1.0.11 silent list coercion) + **DeepSeek V4 MoE tool
injection** (LLM01 with MoE-routing surface) + **TikTok-class agent
hijack** (BlackHat Asia 2026, LLM08).

Plus four new product surfaces: `aak coverage`, `aak pipelock import`,
`aak inspect-ide` (with stdio LSP server + Zed extension), and
`aak parity report` for the runtime `@aak.parity.check` decorator.

### CLI (4 new commands)

- `aak coverage --source ox --format text|json|badge`
- `aak pipelock import <policy.yaml> [--output ...] [--dry-run]`
- `aak inspect-ide [PATH] [--format lsp|json|text] [--serve]`
- `aak parity report --dimension model --metric price [--window 7d]`

### Runtime helpers

- `agent_audit_kit.parity.check` — decorator records (dimension, metric)
  per invocation; `report()` runs the parity assertion.
- `agent_audit_kit.checks.economic_drift.assert_parity` — per-bucket
  mean drift assertion. Raises `ParityDriftError`.
- `agent_audit_kit.sanitizers.deepseek.sanitize_tool_description` —
  strips control + routing-poison tokens, truncates. Suppresses the
  SAST rule when called in the same function.
- `agent_audit_kit.autofix.langgraph_toolnode.fix` — idempotent
  text-level rewrite of `ToolNode([...])` → `ToolNode(tools=[...])`.

### Editor / IDE

- `agent_audit_kit/ide/lsp_diag.py` — minimal stdio LSP server,
  `diagnostics_for(path)` helper.
- `editors/zed/extension.toml` + README — Zed extension that
  auto-launches `agent-audit-kit inspect-ide --serve`.

### Coverage / housekeeping

- `agent_audit_kit/data/ox-cve-manifest.json` — 19 OX-disclosed CVE
  entries, all currently covered.
- `schema/ox-cve-manifest.schema.json` — JSON Schema for the
  manifest.
- `.github/workflows/badge-ox-coverage.yml` — publishes
  `public/badges/ox-coverage.json` automatically.
- `public/corpora/manifest.json` — bumped `schema_version` to `2`;
  every corpus carries `source_url`, `license`, `fetched_at`.
- `agent_audit_kit/corpus/manifest.py` — `CorpusEntry` accepts the
  new provenance fields.

## Upgrade notes

- No breaking changes. `RULE_COUNT` is now `180`; `__version__` is
  `0.3.9`.
- The corpus manifest schema bumped from `1` → `2`. Old AAK readers
  ignore the new fields gracefully.
- Pipelock import emits comments for any unknown top-level keys
  rather than dropping them. Review the output before committing.
