# AgentAuditKit v0.3.8 — release notes

**Released:** 2026-04-27
**Tag:** `v0.3.8`
**Rule count:** 175 (was 169)
**Test count:** 830 (was 810)
**Predecessor:** [v0.3.7](https://github.com/sattyamjjain/agent-audit-kit/releases/tag/v0.3.7) (2026-04-26)

## Headline

**PR-title indirect prompt injection** (CVSS 9.4 Comment-and-Control) +
**MCP function-hijacking** (arXiv 2604.20994, 70-100% ASR) +
**Atlassian MCP RCE chain** (CVE-2026-27825/27826) + **wild IPI corpus**
(2026-04-24, 10 payloads) + **MCPJam vendored fork** (CVE-2026-23744).

Plus four new product surfaces: `aak corpus update`, `aak diff`,
`aak suggest`, VS Code SARIF reader.

## Why one big release

Three high-impact agentic-security disclosures landed in the same
week (2026-04-22 → 2026-04-25). Splitting them across multiple
releases would have cost SLA on the slowest one. v0.3.8 is the
class-coverage ship.

### CLI surfaces

- **`aak corpus update [--ipi|--fhi|--all]`** — out-of-band threat-corpus
  refresh with SHA-256 verification.
- **`aak diff --baseline prev.sarif --current now.sarif`** — SARIF
  baseline diff with `aak_diff_state` per-result tag.
- **`aak suggest <sarif> [--pr]`** — Markdown PR body with per-rule
  remediation hints.

### VS Code extension

- New `vscode-extension/src/sarifReader.ts` — load any SARIF file
  via the `agent-audit-kit.loadSarif` command and see findings as
  inline diagnostics. Layered alongside the existing JSON-on-save
  surface.

### Docs / housekeeping

- `docs/comparison-gitlab-agentic-sast.md` — feature matrix vs.
  GitLab 18.11 Agentic SAST GA (Ultimate-tier paywall).
- `scripts/backfill_cve_property.py` + `tests/test_v0_3_8_features.py`
  fence the docs↔registry CVE consistency invariant.
- `public/corpora/manifest.json` — initial corpus manifest at
  `gh-pages`-eligible path.

## Caveats

- Corpus manifest SHA-256 placeholders fill in on first gh-pages
  publish.
- `aak suggest --apply-trivial` codemods queue for v0.3.9.
- Rust + Java AST passes still regex-only (tracked #22, #23).
