## Why we qualify

### Build provenance + signing

- **PyPI Trusted Publishing (OIDC).** All `agent-audit-kit` wheel + sdist
  uploads to PyPI are issued via GitHub Actions OIDC — no long-lived API
  token in repository secrets. Configured in `.github/workflows/release.yml`.
- **Sigstore attestations.** Every release ships
  `rules.json.sigstore.bundle`, `sbom.cdx.json.sigstore.bundle`, and
  `sbom.spdx.json.sigstore.bundle` produced by the `sigstore-python`
  GitHub Action (keyless OIDC). Verifiable with `cosign verify-blob` or
  `python -m sigstore verify`.
- **SLSA Build provenance v1.** Generated by
  `actions/attest-build-provenance@v4` and attached to every Release
  asset.

### Policy alignment

- **Immutable Action manifest.** `action.yml` ships
  `immutable: true`, aligned to the GitHub Actions 2026 Security Roadmap.
- **All third-party Actions in our own workflows are SHA-pinned.**
  AAK-GHA-IMMUTABLE-001 (shipped in v0.3.4) is the regression fence
  that guarantees this stays true on every PR.

### Quality bar

- **Test count:** 749 tests across 60 files (as of v0.3.4 — bumps each
  release).
- **Type-check:** `mypy --strict` clean on `agent_audit_kit/`.
- **Lint:** `ruff check` clean across `agent_audit_kit/`, `tests/`,
  `scripts/`.
- **Code-scan:** `CodeQL` analysis clean on every PR (configured in
  `.github/workflows/codeql.yml`).
- **CVE response SLA:** publicly committed to 48-hour rule coverage for
  every NVD-disclosed MCP CVE. Audit trail at
  [`CHANGELOG.cves.md`](../../CHANGELOG.cves.md).
