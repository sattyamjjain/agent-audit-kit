# Preset: `mcp-ox-2026-04`

| Field | Value |
|---|---|
| Shipped | v0.3.6 (2026-04-26) |

| Sources | OX Security MCP advisory hub, Cloudflare MCP-defender reframe |

## What it activates

Bundles every AAK rule that maps to the OX MCP STDIO architectural
class so `aak scan --preset mcp-ox-2026-04 .` emits only OX-relevant
findings:

- `AAK-STDIO-001`
- `AAK-WINDSURF-001`
- `AAK-FLOWISE-001`
- `AAK-ANTHROPIC-SDK-001`
- `AAK-MCP-STDIO-CMD-INJ-001` (Python)
- `AAK-MCP-STDIO-CMD-INJ-002` (TypeScript)
- `AAK-MCP-STDIO-CMD-INJ-003` (Java)
- `AAK-MCP-STDIO-CMD-INJ-004` (Rust)
- `AAK-MCP-MARKETPLACE-CONFIG-FETCH-001`
- `AAK-AZURE-MCP-NOAUTH-001`
- `AAK-LMDEPLOY-VL-SSRF-001`
- `AAK-SPLUNK-MCP-TOKEN-LEAK-001`

## CLI usage

```bash
agent-audit-kit scan --preset mcp-ox-2026-04 .
```

## GitHub Action usage

```yaml
- uses: sattyamjjain/agent-audit-kit@v0.3.13
  with:
    preset: mcp-ox-2026-04
```

The `preset:` input unions with `rules:` if both are passed; bare
`preset:` activates only the listed rules.

## Why one preset per incident

OX's April 2026 reframe spans 8 CVEs across 4 SDK languages plus
two adjacent classes (marketplace fetch, Azure server-side missing
auth, Splunk config leak, LMDeploy VL SSRF). Running the full AAK
rule set against a large agent monorepo can take minutes; the preset
narrows the work to "did we catch the OX class?" in seconds.

Future presets follow the same convention: one preset per high-value
incident or vendor stack. See
[`docs/presets/AUTHORING.md`](./AUTHORING.md) (queued for v0.4.0)
for community contribution flow.
