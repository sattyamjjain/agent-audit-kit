    Task A — SLA-watcher dedup (root cause fix)

    scripts/cve_watcher.py currently dedups against CHANGELOG.cves.md. A CVE that sits in the SLA queue with no rule yet never reaches the changelog, so the 6-hourly cron re-opens it. Fix:

    1. Persist seen CVE IDs in .aak/cve-watcher-state.json (write after successful file-issue, never reset).
    2. Cross-check open cve-response issues via the GitHub REST API (no gh) before filing — if a CVE ID appears anywhere in an existing open issue title, skip.
    3. Add scripts/close_duplicate_cve_issues.py that groups open cve-response issues by extracted CVE ID, keeps the lowest-numbered one, closes the rest with a cross-reference body.
    4. Fixture test tests/test_cve_watcher_dedup.py replays the same CVE three times across different timestamps and asserts exactly one issue-file call.

    Task C — AAK-ANTHROPIC-SDK-001 (HIGH)

    New scanner agent_audit_kit/scanners/mcp_sdk_hardening.py. Fires when all three are true:

    1. Repo's manifests declare the upstream Anthropic/MCP SDK:
    2. Python: mcp or modelcontextprotocol in requirements*.txt/pyproject.toml.
    3. TS/JS: @modelcontextprotocol/sdk in package.json.
    4. Java: io.modelcontextprotocol:* in pom.xml/build.gradle.
    5. Rust: mcp or modelcontextprotocol in Cargo.toml.
    6. Repo exposes an MCP server with a STDIO transport — any file contains StdioServerTransport, StdioTransport, stdio_server, StdioServer, etc.
    7. No sanitizer marker is present (all of these opt out):
    8. shlex.quote or shlex.split import used against argv-assembly variables (Python).
    9. An allow-list function (regex matches ALLOWED_, WHITELIST, VALID_TOOLS, etc.).
    10. stdio_disabled: true or transports: ["http"]/["sse"] in a discovered MCP config.
    11. .agent-audit-kit.yml contains accepts_stdio_risk: true with a non-empty justification field (documented risk acceptance).

    Rule fields: owasp_agentic_references=["ASI02","ASI10"], owasp_mcp_references=["MCP01:2025"], incident_references=["OX-MCP-2026-04-15"], aicm_references=["AIS-07","STA-08"]. No CVE — Anthropic declined to CVE.

    Fixtures: tests/fixtures/incidents/ox-mcp-2026-04-15/{python,typescript,java,rust}/ each with vulnerable/ + sanitized/ subdirs. No live exploit code — just minimal reproducible server scaffolds.

    Cross-link: AAK-STDIO-001.description gets a trailing sentence pointing to the SDK-level rule.
